# 009 目標設定機能

## 概要
累計目標・期間目標を作成し、進捗バーで達成率を確認できる機能を実装する。

## 対象フェーズ
Phase 3 — 継続支援機能

## データモデル（参照）
`Goal`（id / type / targetAmount / categoryId / periodStart / periodEnd / achievedAt / createdAt / updatedAt）

## 依存チケット
- 001（DB 基盤）
- 005（ホーム画面の進捗バーエリア）

## タスク

### ドメイン層
- [×] `Goal` エンティティクラス
- [×] 達成判定ロジック（`currentAmount >= targetAmount`）
- [×] 期間目標の対象金額集計ロジック（`periodStart` ～ `periodEnd` の WorkSession を合計）

### インフラ層
- [×] `GoalRepository` インターフェース定義
- [×] `GoalRepositoryImpl`（Drift 実装）
  - [×] アクティブな目標一覧取得（`achievedAt` が null のもの）
  - [×] 目標作成 / 更新 / 削除

### アプリケーション層
- [×] `goalsProvider` — アクティブな目標一覧を返す
- [×] `goalProgressProvider(goalId)` — 達成率（0.0 〜 1.0）を返す
- [×] `GoalNotifier`（作成 / 更新 / 削除 / 達成マーク）
- [×] 達成検知ロジック（セッション保存時に目標達成をチェックし、`achievedAt` を記録）

### プレゼンテーション層

#### 目標一覧（設定画面の子ページ）
- [×] アクティブな目標カード（目標名 / 種別 / 目標金額 / 進捗バー / 現在金額）
- [×] 達成済み目標の折りたたみセクション

#### 目標作成 / 編集フォーム
- [×] 目標種別選択（累計 / 期間）
- [×] 目標金額入力
- [×] 対象カテゴリ選択（任意。未選択 = 全カテゴリ合計）
- [×] 期間目標の場合：開始日・終了日ピッカー
- [×] 保存 / キャンセルボタン

#### ホーム画面との連携（005 の進捗バーエリア）
- [×] `goalProgressProvider` を使って進捗バーを描画

## 完了条件
- 目標の作成・編集・削除が動作する
- 進捗バーが正しい達成率を表示する
- セッション保存後に達成判定が走り `achievedAt` が記録される
- `flutter analyze` がエラーなし
