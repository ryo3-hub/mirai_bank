# 004 手動入力方式 作業記録

## 概要
過去の作業時間を後から手動入力し、WorkSession として保存する機能を実装する。
既存セッションの編集・削除も同じフォームを使う。

## 対象フェーズ
Phase 1 — MVP コア

## データモデル（参照）
`WorkSession`（inputMethod = `manual`）

## タスク

### アプリケーション層
- [×] `ManualRecordNotifier`
  - [×] `create(categoryId, date, durationSec, memo)` — amount を計算して保存
  - [×] `update(sessionId, ...)` — updatedAt を更新して上書き保存
  - [×] `delete(sessionId)` — ソフトデリート

### プレゼンテーション層
- [×] 手動入力フォーム画面 / ボトムシート（新規 & 編集共用）
  - [×] カテゴリ選択（ドロップダウン or 選択ダイアログ）
  - [×] 日付選択（`showDatePicker`、過去日・今日を選択可）
  - [×] 時間入力（時 / 分 をそれぞれ NumberPicker or テキストフィールド）
  - [×] メモ入力（任意）
  - [×] 保存ボタン（バリデーション：カテゴリ必須、時間 > 0）
  - [×] キャンセルボタン
  - [×] 編集時は「削除」ボタンも表示（確認ダイアログ付き）
- [×] 履歴一覧 / ホーム画面から手動入力フォームへの遷移導線

### バリデーション
- [×] カテゴリ未選択時はエラー表示
- [×] 時間が 0 分以下の場合はエラー表示
- [×] 未来日付は入力不可（今日が上限）

## 完了条件
- 手動入力でセッションが作成され、履歴に正しく反映される
- 編集・削除が動作する
- `flutter analyze` がエラーなし
