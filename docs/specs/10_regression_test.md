# リグレッションテスト仕様書

リリース前に通すマニュアルチェックリスト。過去 issue で直したバグの再発と、
リファクタによる挙動デグレを検出する目的。**iOS / Android の両方** で
リリースビルドを使い、一通り通す（目安: 30 分）。

## 使い方
- 行頭 `- [ ]` を `- [×]` に書き換えながら進める（CLAUDE.md の Todo 表記に準拠）
- 各項目に `→ 期待結果` が書いてあるので、画面の見た目と挙動の両方を確認
- 関連の挙動詳細は各画面の spec（[01〜09](./README.md)）を参照
- 失敗した項目は GitHub issue を切ってラベル `regression` を付ける

## 前提
- [ ] アプリを **完全アンインストール → 再インストール** した素の状態で開始（オンボーディング条件 #1 を再現するため）
- [ ] テスト端末の OS バージョン: iOS 16+ / Android 8+（min_sdk 21）
- [ ] 端末の言語が日本語、タイムゾーンが Asia/Tokyo
- [ ] **Light / Dark の両モード** で表示を切り替えながら一通り見る（OS の外観設定で切替）

## 1. オンボーディング初回起動（[01_onboarding.md](./01_onboarding.md)）

### イントロステップ
- [ ] アプリ起動 → イントロ画面 → 「はじめる」が表示される
- [ ] 「はじめる」タップ → カテゴリ設定ステップへ遷移
- [ ] イントロにスキップ動線が無い

### カテゴリ設定ステップ
- [ ] **プリセット** モードがデフォルトで選択されている
- [ ] プリセット未選択で「始める」→ エラートースト「プリセットを選んでください」
- [ ] `CategoryMasterPickerSheet` で大カテゴリ → 小カテゴリの 2 段を選べる
- [ ] **カスタム** モード切替 → カテゴリ名 / 時給の入力欄が出る
- [ ] カテゴリ名空 → 「カテゴリ名を入力してください」
- [ ] カテゴリ名 31 文字以上 → 「30文字以内で入力してください」
- [ ] 時給 99 → 「100円以上を入力してください」
- [ ] 時給 10001 → 「10000円以下を入力してください」
- [ ] アイコン 12 種・カラー 8 種が選択可能
- [ ] **「あとで設定する」**タップ → ホームに遷移し、タイマーカードに「カテゴリを追加」が出る（以降ホーム）

### 目標設定ステップ（カテゴリ作成時のみ）
- [ ] Q1 / Q2 / Q3 が順に出る（進捗インジケータ 1/3 → 2/3 → 3/3）
- [ ] 各質問におすすめバッジが 1 つだけ付く（平日中心 / 短時間でテンポよく / 中期的に身につけたい）
- [ ] 結果画面に累計金額・月あたり金額・達成予定日が出る
- [ ] ハードコンボ（毎日 × じっくり × 1年など）でオレンジのヒントカードが出る
- [ ] Q1 の戻るボタン → カテゴリがソフトデリートされてカテゴリステップに戻る
- [ ] Q2/Q3 の戻るボタン → 前の質問に戻る（カテゴリは保持）
- [ ] 「あとで設定する」→ ホームへ

### リマインダー曜日選択ステップ（目標設定時のみ）
- [ ] 初期値で 7 曜日すべて選択済み
- [ ] 初期時刻が 21:00
- [ ] 時刻行タップで `MiraiTimePickerSheet` が開く（ドラムロール）
- [ ] 全曜日を解除して「設定する」→ エラートースト
- [ ] 「リマインダーを設定する」→ OS の通知許可ダイアログ → 許可後ホーム
- [ ] 「あとで設定する」→ ホーム（リマインダー無効のまま）

## 2. ホーム（[02_home.md](./02_home.md)）

### TodayAmountCard
- [ ] 今日の積み上げ金額が大きく表示される（48px、primary 色）
- [ ] 今日の作業時間が `Xh Ym` 形式で表示される
- [ ] 連続学習 0 日 → フッターに 🔥 行は出ず、ひとことメッセージのみ
- [ ] 連続学習 1+ 日 → `Icons.local_fire_department` + 「N 日連続」
- [ ] 連続学習 3/7/30/100/365 日ぴったり → 節目ラベル付き（「3日達成！」等）
- [ ] **アプリ再起動ごとに** ひとことメッセージが（同ティア内で）別文言になる

### Timer card
- [ ] **idle**：カテゴリ選択カード + プリセット最大 3 枚 + 「N 分で開始」ボタン
- [ ] プリセット 0 件のとき：「プリセットを追加」リンクが出る
- [ ] プリセット 4 件以上：先頭 3 件だけホームに出る
- [ ] カテゴリ 0 件状態 → 「カテゴリがありません」+ 「カテゴリを追加」ボタン
- [ ] **running**：残り時間が `mm:ss` で表示（56px、tabular figures）
- [ ] 一時停止中 → 残り時間がグレーアウト、再開ボタンが GitHub Green
- [ ] 4:59 で停止 → 「5 分未満だったので記録しませんでした」+ WorkSession 未作成
- [ ] 5:00 で停止 → セッション保存 + `AmountFlash` 演出（時給 2000 円なら 167 円）
- [ ] **目標時間到達 → 自動停止**：セッション保存 + プッシュ通知

### DashboardGoalsSection
- [ ] 目標 0 件 → 「目標を追加」カードが表示される
- [ ] 目標 1〜5 件 → 各目標カードが表示される（進捗バー / % / 金額）
- [ ] 目標 6 件以上 → 5 件表示 + 「他 N 件」リンクで `/settings/goals` へ

### バックグラウンド復帰
- [ ] タイマー開始 → アプリを Home に戻す → 30 秒待ってから復帰 → 経過時間が進んでいる
- [ ] タイマー開始 → アプリを **完全終了（タスクキル）** → 再起動 → タイマー継続中

## 6. 設定（[06_settings.md](./06_settings.md)）

### 設定一覧画面
- [ ] 「管理」セクション：カテゴリ管理 / 目標設定 / 履歴 / タイマープリセット / データ管理
- [ ] 「通知」セクション：通知設定
- [ ] 「その他」セクション：お問い合わせ / アプリを評価する / 利用規約 / プライバシーポリシー / アプリについて

### データ管理
- [ ] エクスポート → 共有シートで `mirai_bank_backup_YYYYMMDD_HHmm.json` を保存できる
- [ ] エクスポート JSON を **同一バージョン** のアプリで復元 → 確認ダイアログ → 全件上書き
- [ ] 古い schemaVersion の JSON で復元 → エラートースト（`BackupFormatException`）

### 通知設定
- [ ] リマインダー OFF → 時刻 / 曜日ピッカーがグレーアウト
- [ ] リマインダー ON → 時刻と曜日が編集可能
- [ ] 曜日を 0 件にすると赤い警告文「曜日を 1 つ以上選んでください（現在は通知されません）」が出る
- [ ] 達成通知 ON/OFF が即時に DB に保存される
- [ ] 指定時刻 + 曜日に **実機で通知が来る**（テストは 2〜3 分先の時刻で）

### お問い合わせ
- [ ] タップ → 既定メールクライアントが立ち上がる
- [ ] 件名「[mirai_bank] お問い合わせ」、本文末尾にアプリバージョン / OS / 端末モデルが入る

### アプリを評価する
- [ ] タップ → iOS なら SKStoreReviewController、Android なら Google Play In-App Review

### 法務文書
- [ ] 利用規約 / プライバシーポリシーがアプリ内で表示される
- [ ] OSS ライセンス（アプリについて → ライセンス）が `showLicensePage` で開く

## 10. 横断項目

### テーマ
- [ ] OS のダーク / ライト切替 → アプリも追従（Sky 系の primary、夜間カードの色）
- [ ] AppBar / FAB / SegmentedButton がテーマ色（Sky `#0EA5E9`）で描画
- [ ] 削除ボタンは `colorScheme.error` の赤フィルド

### ボトムシート共通
- [ ] すべてのシートが **シート外タップ / 下方向スワイプで閉じる**（`isDismissible: true`）
- [ ] 専用「キャンセル」ボタンが無い

### トースト
- [ ] すべての通知が画面 **上部** から `TopToast` で出る（`SnackBar` は使わない）
- [ ] エラートーストが赤系で出る

### スプラッシュ / アイコン
- [ ] アプリ起動時のスプラッシュ画像が正しく表示される（ライト / ダーク両モード）
- [ ] ホーム画面のアイコン画像がデザイン通り

### Sentry
- [ ] リリースビルド（`--dart-define=SENTRY_DSN=...` 付き）でクラッシュが Sentry に届く
- [ ] DSN なしビルドでは Sentry 初期化を完全スキップ（ログにエラーが出ない）

## リリースゲート（最低限ここは合格）
ストア配布前に以下が全部 `[×]` になっていること：

- [ ] オンボーディングをスキップ無し / 全スキップ / 途中スキップの 3 通り通す
- [ ] タイマーを 5 分以上 / 5 分未満 / 自動完了 の 3 通り
- [ ] 履歴のスワイプ削除が動く
- [ ] エクスポート → 同一バージョンでインポートが動く
- [ ] 通知が指定曜日 / 時刻に届く（実機で 2〜3 分先に設定して確認）
- [ ] ダークモードで全画面が崩れない
